2. Comparing Three Methodologies

🏆

PatchCore

Main architecture — Implemented from scratch

  • Patch-level feature matching.
  • Custom ResNet18 backbone.
  • Utilizes Memory Bank + Coreset Subsampling.
  • k-NN scoring (Custom metric).
  • Provides Pixel-level Anomaly Heatmaps.
🔄

Autoencoder

Reconstruction Error algorithm

  • Encode and Decode architecture.
  • Trained to reconstruct normal images.
  • Uses MSE loss for reconstruction scoring.
  • Simple and highly interpretable.
🔍

CNN + OC-SVM

Classical ML integration

  • Uses ResNet18 for Feature Extraction (512-d).
  • One-Class SVM creates a decision boundary.
  • Anomaly detection based on boundary distance.
  • Fast inference, efficient with low data requirements.
  • No heatmap localization (Image-level only).

3. PatchCore Core Operations

Why build from scratch?

  • No reliance on heavy external compiled libraries like sklearn.
  • Takes full advantage of CUDA Tensor Operations.
  • Enables bespoke low-memory hashing PyTorchLSHIndex.

Technical Merit

  • Multi-scale fusion (Layer 2 & Layer 3) preserves spatial metrics.
  • Random Projection aggressively drops distances from 384-D down to 128-D.
  • Greedy Coreset discards 90% of redundant nominal patches.

Speed Breakthrough: Deploys Random Projection & torch.cdist collapsing 384-D -> 128-D dimensions, skyrockets calculation speed by 3x.
Statistical Threshold: Computes a rigorous 99.5th Percentile mathematical boundary natively discarding False Positives.

SOTA Architecture Core: LSH & Hardware

Multi-probe Locality-Sensitive Hashing

O(1) rapid lookup fetching instead of heavy O(N).

  • Mechanism: Divides vectors into 4096 individual Buckets (12-bits).
  • Optimization: Proximity search via XOR (^) bit-flips.
  • Efficiency: Analyzes all contiguous Hamming Distance = 1 arrays.
  • Impact: Eliminates fetch latency, shifting pipeline output from 5 FPS to 35 FPS.

Hardware Engine Configurations

100% On-premise resources. Zero external Cloud AI APIs.

  • Capacity: ResNet18 backbone: Exactly ~11 Million Parameters (11M Params).
  • Inference: Driven by NVIDIA GTX 1660 Ti (6GB Private VRAM).
  • Pipeline: Full model pre-loading to CUDA; bypasses PCI-e bottlenecks.
  • Portability: Production-ready; ONNX-exported for Edge C++ integration.

Knowledge Distillation (KD)

🎓 Teacher (FROZEN Pretrained)

  • Standard ResNet18 trained on ImageNet knows how to perfectly extract shapes.
  • Absolutely frozen (No Gradients) allowing it to act as an Oracle.

📚 Student (LEARNABLE Custom)

  • A lightweight CustomResNet, dramatically saving Memory VRAM.
  • Starts completely from scratch (No internal weights whatsoever).

Strategy Loss = Σ MSE(Student_L[i], Teacher_L[i])

By executing Mean Squared Error computation across 4 architectural bottlenecks, the Student forces its vectors to mimic the exact spatial logic of the Teacher.

  • Teacher Bias: The Student's theoretical upper bound is strictly locked to the Teacher's capability. If the Teacher fails, the Student fails.

Autoencoder Reconstruction Architecture

Anomaly Detection Mechanism

The model only learns to reconstruct perfect objects. When interpreting an anomaly, it hits massive loss trying to render it perfectly. The pixel differential difference between reality and reconstruction generates a Glowing Error Heatmap.

Encoder (Compression)

5 Convolution blocks squeeze the nominal RGB signals into a spatial bottleneck holding only 4096 dynamic scalars.

Decoder (Upsampling Recon)

5 ConvTranspose block stretch back to 256x256. Final layer drops normalization and ReLU limiting to expose raw RGB spectrum outputs.

📊 In-depth Analysis

This graph displays the unsupervised Autoencoder training over 50 epochs, tracking MSE Loss alongside model Validation Accuracy (AUROC).

  • ✔️ Symmetrical Convergence: A beautiful inverse correlation is observed. As Training/Valid Loss minimizes, the Accuracy curve smoothly climbs and stabilizes at ~97.5%.
  • ✔️ No Overfitting: The Validation Loss tightly hugs the Training Loss throughout all 50 epochs without any upward bounce, confirming excellent generalization.
  • ⚠️ Bootstrapping Delay: Because the model learns latent features from scratch (unlike Knowledge Distillation), it requires ~15 epochs just to bootstrap basic structural representations before accuracy steeply rises.
  • ⚠️ Performance Plateau: Beyond Epoch 35, the Accuracy strictly plateaus despite minor loss improvements. Extending training further wastes compute.

🎯 Individual Metric Evaluation

  • Precision (92.6%): Exceptionally high. When the AI flags an anomaly, it is almost certainly a defect. This drastically minimizes False Alarms, saving human review time.
  • Recall (70.0%): The system misses 30% of subtle anomalies (False Negatives). This indicates a conservative mathematical threshold for flagging defects.
  • Specificity (85.0%): Strong ability to correctly identify Normal items, ensuring perfectly clean products aren't mistakenly scrapped.
  • Accuracy (74.0%): Shows overall correctness but is fundamentally skewed by the dataset's class imbalance (1,258 NG vs 467 OK).

⚖️ Mutual Support & Trade-offs

  • Precision vs. Recall Trade-off: The current configuration heavily prioritizes Precision over Recall. In real industrial environments, if escaping defects is unacceptable, we must manually lower the anomaly threshold to boost Recall (catching more defects) at the acceptable cost of reducing Precision (more False Alarms).
  • The Anchoring Role of F1-Score (79.7%): Because standalone Accuracy is misleading due to class imbalance, the F1-Score acts as the ultimate balancing metric. It mathematically harmonizes the stark difference between Precision (92.6%) and Recall (70.0%), providing a single, reliable source of truth for the model's true capability.

In real-world industrial assembly lines, the ability to generate a precise Localization Heatmap is absolutely mandatory for engineers to visually verify and root-cause anomalies. OC-SVM's inability to localize makes it unsuitable for automated visual QA.

7. Project Conclusion & Future Outlook

✅ Key Achievements

  • SOTA Architecture Mastery: Rebuilt PatchCore from scratch using Custom PyTorch, bypassing black-box library dependencies.
  • Hardware Optimization: Achieved ~121ms ultra-low latency inference via VRAM pre-loading and LSH (Locality-Sensitive Hashing).

🚀 Future Trajectories

  • Hardware Integration: Direct connection to industrial GigE vision cameras (e.g., Basler) on an active factory conveyor belt.
  • Edge Deployment: Compress models via TensorRT / ONNX for deployment on low-power edge IoT devices (e.g., Jetson Nano).
